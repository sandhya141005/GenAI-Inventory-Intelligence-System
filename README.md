# GenAI Inventory Intelligence Platform

Production-grade inventory decision intelligence system.

Flow:

```text
PostgreSQL -> FastAPI Decision Engine -> REST APIs -> Next.js Dashboard
```

The analytics dashboard is backed by PostgreSQL data through deterministic business rules. No dashboard metrics are generated by an LLM.

## Services

- `backend/` - FastAPI, SQLAlchemy, Alembic, PostgreSQL analytics endpoints
- `app/`, `components/`, `lib/` - Next.js dashboard consuming `NEXT_PUBLIC_API_URL`

## Backend

```bash
cd backend
docker compose up --build -d
```

The API uses:

```text
postgresql+psycopg2://inventory:inventory@postgres:5432/inventory_copilot
```

Analytics endpoints:

- `GET /api/analytics/overview`
- `GET /api/analytics/inventory`
- `GET /api/analytics/recommendations`
- `GET /api/analytics/reports`
- `GET /api/analytics/charts`
- `GET /api/analytics/transfers`
- `GET /api/analytics/inventory-aging`
- `GET /api/analytics/notices`

## Frontend

```bash
npm install
$env:NEXT_PUBLIC_API_URL="http://localhost:8000"
npm run dev
```

Then open `http://localhost:3000`.

## Required Data Tables

The decision engine reads these tables only for analytics:

- `products`
- `stores`
- `sales`
- `warehouse_stock`
- `transfers`

Run Alembic migrations before using the analytics endpoints in a fresh database.
